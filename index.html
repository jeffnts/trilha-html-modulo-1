A fascinante história da Web

A história da computação é uma jornada incrível, desde máquinas gigantescas que ocupavam salas inteiras até os dispositivos que carregamos no bolso. Tudo começou com o ENIAC em 1945, considerado o primeiro computador eletrônico de grande escala. A evolução foi rápida, passando pelos mainframes, minicomputadores e, finalmente, pelos computadores pessoais (PCs) que popularizaram a tecnologia.

Paralelamente, a internet surgia como um projeto militar chamado ARPANET. A grande virada para nós, desenvolvedores, foi a criação da World Wide Web (WWW) por Tim Berners-Lee em 1989. Ele desenvolveu o HTML, o HTTP e o primeiro navegador. Isso transformou a internet de uma rede de acadêmicos para a plataforma global que conhecemos hoje.

"A Web como eu a concebi, não a desenvolvemos para ser uma coisa para se olhar, mas um meio para se trabalhar em conjunto." - Tim Berners-Lee

O que são Clients?

No contexto da web, um cliente (client) é o lado do usuário na equação. É o software ou dispositivo que faz uma requisição de informações a um servidor. O exemplo mais comum é o seu navegador de internet (como Chrome, Firefox, Safari). Quando você digita um endereço e pressiona Enter, seu navegador (o cliente) envia um pedido para o servidor daquele site.

Outros exemplos de clientes incluem aplicativos de celular, clientes de e-mail (como o Outlook) e até mesmo outros sistemas que consomem APIs. A função principal do cliente é iniciar a comunicação e exibir a resposta recebida do servidor de forma compreensível para o usuário.

O que são Servers?

Se o cliente pede, o servidor (server) é quem atende. Um servidor é um computador (ou um sistema de computadores) robusto, sempre conectado à internet, cuja função é armazenar, processar e entregar os dados solicitados pelos clientes.

Quando o servidor do site da DIO recebe uma requisição do seu navegador, ele localiza os arquivos HTML, CSS e JavaScript, processa qualquer lógica necessária (como verificar se você está logado) e envia tudo de volta para o seu navegador montar a página. O conceito de servidor era apenas para computadores enormes, mas hoje qualquer computador pode atuar como um servidor, embora para sites de grande tráfego sejam usadas máquinas especializadas.

Estrutura básica de um documento HTML

Todo documento HTML segue uma estrutura padrão, que informa ao navegador como interpretar e renderizar o conteúdo. É como o esqueleto da página.

Tags Essenciais

A estrutura fundamental é composta por: <!DOCTYPE html>, <html>, <head> e <body>.

  • <!DOCTYPE html>: Define a versão do HTML (neste caso, HTML5).
  • <html>: A raiz de todo o documento.
  • <head>: Contém metadados, título da página, links para CSS e scripts. Não é visível no corpo da página.
  • <body>: Onde todo o conteúdo visível (textos, imagens, links) é colocado.

Antigamente, para definir uma cor de texto, usava-se uma tag como a tag font. Por exemplo: este texto ficaria azul.

Listas Ordenadas e Não Ordenadas

Listas são fundamentais para organizar informações em HTML. Existem dois tipos principais, e a escolha depende se a ordem dos itens importa ou não.

Exemplo de Lista Não Ordenada (<ul>)

Usada quando a sequência dos itens não é relevante. Pense em uma lista de compras.

  • Café
  • Açúcar
  • Leite (Composição: H2O + outros componentes)
Exemplo de Lista Ordenada (<ol>)

Usada quando a sequência é crucial, como em um passo a passo de uma receita ou um ranking.

  1. Ferver a água (a 100oC em condições normais).
  2. Adicionar o pó de café.
  3. Coar.
Observação:

Dentro de cada lista (<ul> ou <ol>), cada item é definido pela tag <li> (list item).
